# Design Tokens Guide

Use this page when you need to change brand colours, radii, or shadows without hunting through component files.

## Edit Location

Primary token file: `src/tokens/design-tokens.json`

Supporting token usage:

- `packages/ui/src/styles/globals.css`
- `packages/ui/tailwind.config.ts`

## Editing Rules

1. Change storefront-facing values in `design-tokens.json` first.
2. Keep Shopify-aligned values as the default unless there is a documented app exception.
3. If a token affects Tailwind classes or CSS variables, update the matching config or stylesheet in the same change.
4. Refresh the relevant Storybook docs and stories when the visual meaning changes.

## High-Signal Token Groups

| Group | What it controls |
| --- | --- |
| `colors.brand` | Brand blue, CTA purple, hover state, lime accent |
| `colors.storefront` | Off-white surfaces, pill backgrounds, slider and attention colours |
| `colors.ui` | Shared React semantic mapping for components |
| `typography.fontSize` | Shared scale plus storefront-specific body, label, pill, and input sizes |
| `borderRadius` | Modal, pill, input, textarea, CTA, and submit radii |
| `shadows` | Storefront modal elevation and focus treatment |

## Change Checklist

- Update the token in `src/tokens/design-tokens.json`
- Update matching CSS variables or Tailwind values if required
- Check `Documentation/Design Tokens`
- Check affected storefront stories such as `ProductCard`, `Modal`, `Pill`, and `QuoteDetailsForm`
- Run `pnpm type-check`, `pnpm lint`, and `pnpm build`

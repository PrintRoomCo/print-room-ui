# Print Room UI Storybook

This Storybook is the working source of truth for Print Room UI across three reference surfaces:

- `print-room-ui`: shared React components and Storybook
- `print-room-studio`: production React apps and the design tool
- `print-room-no-design-tool`: production Shopify storefront patterns and content

## What This Storybook Covers

| Surface | Coverage |
| --- | --- |
| `Primitives/*` | Shared UI building blocks such as `Button`, `Input`, `Textarea`, `Dropdown`, `Checkbox`, `RadioGroup`, and `Tabs` |
| `Storefront/*` | Shopify-aligned components and patterns such as `ProductCard`, `Pill`, `BrandingServiceCard`, `FreeShippingProgress`, `QuoteDetailsForm`, `Breadcrumbs`, and `Pagination` |
| `App/Design Tool/*` | Isolated design-tool interactions such as `AnimatedColorButton`, `ColorSwatches`, `ViewTabs`, `MethodCard`, `TechniquesGuideModal`, and `QuantitySummary` |
| `Documentation/*` | Tokens, contribution guidance, hooks, and utility docs |

## Alignment Rules

1. Stories use production-shaped content from Shopify and `print-room-studio`.
2. Shopify storefront tokens are the default brand language for Storybook.
3. App-specific exceptions stay documented instead of silently drifting.
4. Vendored design-tool components stay self-contained and must not import app aliases such as `@/types` or `@/hooks`.

## Using This Library

- Start with the storefront stories when validating customer-facing UI.
- Use the design-tool stories to verify isolated interactions before wiring them into app state.
- Check `Documentation/Design Tokens` before changing colour, radius, or shadow values.

# Design Tokens

Shopify storefront tokens are the default visual source of truth for Print Room brand-facing UI. The shared React library still exposes semantic UI tokens for app surfaces, but those semantics now map back to the live storefront palette first.

Token source file: `src/tokens/design-tokens.json`

## Radius and Shadow Tokens

| Token | Value | Usage |
| --- | --- | --- |
| `borderRadius.modal` | `27px` | Shopify quote and service modals |
| `borderRadius.pill` | `9999px` | Service pills and compact chips |
| `borderRadius.input` | `9999px` | Storefront text inputs and selects |
| `borderRadius.textarea` | `10px` | Quote notes and address fields |
| `borderRadius.cta` | `40px` | Prominent CTA buttons |
| `borderRadius.chip` | `24px` | Compact chip buttons (e.g. favorites) |
| `borderRadius.popover` | `16px` | Popover and drawer panels |
| `shadows.modal` | `0 4px 13px 2px rgba(0, 0, 0, 0.1)` | Storefront modal surface |
| `shadows.popover` | `0 10px 25px rgba(0, 0, 0, 0.1)` | Popover drop shadow |
| `shadows.sliderThumb` | `0 2px 6px rgba(0, 0, 0, 0.15)` | Budget/size slider thumb |
| `shadows.focus` | `0 0 0 3px rgba(43, 57, 144, 0.1)` | Focus ring treatment for inputs |

| `borders.input` | `1px solid black` | Text inputs and selects |

| `borders.chip` | `0.4px solid black` | Compact chip buttons |

## Sizing Tokens

| Token | Value | Usage |
| --- | --- | --- |
| `sizing.pillHeight` | `23px` | Shopify pill button height |
| `sizing.submitHeight` | `70px` | Quote submit button height |
| `sizing.inputMinHeight` | `2.75rem` | Minimum input field height |
| `sizing.sliderThumbSize` | `24px` | Budget slider thumb diameter |
| `sizing.ctaWidth` | `153px` | Standard CTA button width |
| `sizing.ctaHeight` | `47px` | Standard CTA button height |

## App Exceptions

- Internal app surfaces can still override semantic tokens such as `--background`, `--primary`, and `--radius`.
- Storybook documents those overrides as exceptions; they should not silently replace the storefront defaults.
